## Course Description

Students will learn to design and build full-stack web applications using modern frameworks. Topics include HTTP fundamentals, RESTful API design, front-end frameworks (React), back-end frameworks (Node/Express), relational databases, authentication, and deployment.

## Prerequisites

CSE 220 (Software Engineering Fundamentals) or instructor permission.

## Learning Outcomes

- Design and implement RESTful APIs
- Build responsive front-end interfaces using React
- Integrate a relational database with an application backend
- Implement secure authentication and session management
- Deploy an application to a cloud hosting provider

## Required Materials

No textbook required. All readings will be posted on Canvas. Students need a laptop capable of running Node.js and a local database instance (SQLite is sufficient).

## Assignment Submission

Labs are submitted through Canvas as a link to your GitHub repository at the tagged commit for that week (e.g., `lab-03-submission`). Late labs are not accepted; your lowest two lab scores are dropped automatically.

## Exams

There is no traditional written exam in this course. Instead, students complete a **practical coding assessment** during Week 9, held in the lab under proctored conditions, in which they must implement a small API endpoint from a spec within 90 minutes.

## Project Guidelines

The semester project is completed individually. It must include:

- A documented REST API with at least 5 endpoints
- A React frontend consuming that API
- User authentication (JWT or session-based)
- Deployment to a live URL (Render, Vercel, or Railway are acceptable)

## Presentations

Final project demos are held during finals week in 8-minute slots. You will share your screen and walk through your application live — no slides required, but a one-page architecture diagram must be submitted beforehand.
