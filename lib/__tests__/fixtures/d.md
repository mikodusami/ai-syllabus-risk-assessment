## Course Description

An introduction to the design, implementation, and use of relational database management systems. Topics include relational algebra, SQL, normalization, indexing, transactions, concurrency control, and an introduction to NoSQL systems.

## Prerequisites

CSE 150 (Data Structures)

## Learning Outcomes

- Design normalized relational schemas
- Write complex SQL queries including joins, subqueries, and window functions
- Understand indexing strategies and query optimization
- Explain ACID properties and transaction isolation levels
- Compare relational and NoSQL data models

## Required Materials

_Database System Concepts_ (7th ed.) by Silberschatz, Korth, and Sudarshan. PostgreSQL will be used for all assignments; installation instructions are posted on Canvas.

## Assignment Submission

Homework and labs are submitted via Gradescope. SQL scripts must be named according to the convention `lastname_labN.sql` and include comments explaining each query.

## Exam Policy

The midterm and final exams are closed-book, closed-notes, and administered in-class. Calculators are not permitted. A comprehensive review session will be held the week before each exam.

## Final Project

Students work in pairs to design a normalized database for a domain of their choosing (subject to instructor approval), implement it in PostgreSQL, populate it with realistic sample data, and write a report covering their design decisions. A short in-class demo (5 minutes) is required during the last week of class.
