## Course Description

This course introduces the principles and practices of modern software engineering, including requirements analysis, design patterns, version control, testing methodologies, agile development, and software project management. Students will work individually and in teams to design, build, and maintain a mid-sized software application over the course of the semester.

## Prerequisites

CSE 150 (Data Structures) and CSE 160 (Discrete Mathematics), both with a grade of C or better.

## Learning Outcomes

By the end of this course, students will be able to:

- Apply object-oriented design principles to real-world problems.
- Use Git and GitHub for collaborative version control.
- Write and execute unit, integration, and regression tests.
- Participate effectively in an Agile/Scrum development cycle.
- Communicate technical decisions clearly in written and oral form.

## Required Materials

- _Clean Code_ by Robert C. Martin (required)
- GitHub account (free tier is sufficient)
- Access to a Unix-like development environment (WSL, macOS, or Linux)

## Assignment Submission Policy

All homework must be submitted via **Gradescope** as a single PDF or ZIP file by 11:59 PM on the due date. Code submissions must include a `README.md` describing how to build and run the project. Submissions via email will not be accepted under any circumstances.

## Late Policy

- 0–24 hours late: 10% deduction
- 24–48 hours late: 25% deduction
- Beyond 48 hours: not accepted, except in documented emergencies

## Exam Policy

The midterm exam will be closed-book, closed-notes, administered in-class. One handwritten 3x5 note card is permitted. Makeup exams are only granted with prior instructor approval and documented cause (medical, family emergency, university-sanctioned travel).

## Team Project Guidelines

Students will form teams of 3–4 by Week 3. Each team must:

- Submit a project proposal (Week 4)
- Submit two milestone check-ins (Weeks 8 and 12)
- Maintain a public or instructor-visible GitHub repository with meaningful commit history
- Deliver a final working application with test coverage above 60%

## Final Presentations

Each team will present their project during the final two class sessions (Week 15). Presentations are 10 minutes plus 5 minutes of Q&A. All team members must speak. Slides are due 24 hours before your presentation slot via Canvas.

## Academic Integrity

All homework must represent your own individual work. Team projects must credit all contributing members. Any use of another student's code without attribution constitutes a violation of the university's academic integrity policy and will be referred to the Office of Student Conduct.

## AI Usage Policy

Students may use AI coding assistants (e.g., Copilot, ChatGPT) to assist with debugging and boilerplate code. AI-generated code used in a submission must be disclosed in the README under an "AI Assistance" heading. Undisclosed AI-generated submissions will be treated as an academic integrity violation.

## Accommodations

Students requiring accommodations should contact the Office of Disability Services within the first two weeks of the semester and provide documentation to the instructor.

## Attendance

Attendance is not directly graded but is strongly correlated with project team performance. Three or more unexcused absences may result in a participation grade reduction.
